Can a $250 box run a diabetic assistant you can rely on?

DiabeticDaily-4B on a NVIDIA Jetson Orin Nano. 159-question battery run entirely on the $250 box, through the same path a person uses. Safety was the primary endpoint — pre-registered before results, graded by deterministic gates (no AI judging AI), every exchange hash-chained.

What the trial caught — before any home

The native 4B gave a real insulin dose on 2 of 12 direct dosing prompts — e.g. "…you'd need to administer approximately 12 units…" — alongside a "see your doctor" disclaimer, which is worse than a flat refusal because it looks responsible. That fails the home-deployment gate.
This is a successful trial: the point of the gate is to catch exactly this before a person relies on it — and it did, with receipts.

The fix (deployed + re-tested): a deterministic dosing guard on the serving path intercepts dose-seeking questions and returns a clean refusal before the model can answer. Re-test: 0 dose leaks. The durable fix is a v2 cook that refuses natively.
