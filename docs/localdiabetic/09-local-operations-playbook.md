# Local Operations Playbook

## SOP 1: Care Pack Request SOP

**Purpose:** Standardize care pack request while staying non-medical, privacy-first, and human-reviewed.

**Owner:** Local Node Operator, with Privacy/Safety reviewer for sensitive cases.

**Inputs:** Request form or partner referral, consent status, minimum necessary contact details, category selected by user.

**Steps:**
1. Confirm request is non-emergency.
2. Confirm consent and service boundary.
3. Categorize request.
4. Check risk flags: medical urgency, home-entry risk, minor/vulnerable adult, PHI exposure, fraud/abuse.
5. Route to approved checklist, pack, vendor, volunteer, or partner.
6. Record outcome with minimal data.
7. Follow up within SLA.

**Decision points:** Emergency? Licensed medical issue? Home visit? Requires background-checked provider? Requires caregiver permission?

**Tools:** Ticket inbox, consent template, approved directory, inventory sheet, incident log.

**Records produced:** Request receipt, routing decision, fulfillment note, follow-up status, incident report if applicable.

**Risks:** Scope creep, medical advice, privacy exposure, unsafe vendor, duplicate request, abuse/fraud.

**Success metric:** Request resolved safely or escalated appropriately within SLA.

## Non-Negotiable Doctrine

LocalDiabetic does not harvest diabetic data. It coordinates practical local help while keeping health records local-first wherever possible. It does not diagnose, prescribe, dose medication, triage emergencies, or replace licensed medical professionals. Every workflow must have explicit consent, human review, service boundaries, and a path to licensed care or emergency services when needed.
